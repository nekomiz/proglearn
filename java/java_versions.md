## JDK 1.0 (1996)
Первая версия языка программирования Java была выпущена под названием **Oak** (Дуб), но позже переименована в Java. Основные особенности:
- Введение основных синтаксических конструкций языка, таких как классы, методы, интерфейсы, пакеты.
- Поддержка многопоточности.
- Сборщик мусора (Garbage Collector).
- Первая реализация виртуальной машины Java (JVM).
  
## JDK 1.1 (1997)
Основные изменения:
- Добавление внутреннего класса (inner class).
- Появление рефлексии – возможности динамического доступа к классам и методам.
- Реализация коллекции классов (`java.util.Collection`).
- Внедрение механизма сериализации объектов.

## J2SE 1.2 (1998)
Это был значительный релиз, который стал известен как «Java 2». Основные нововведения:
- Полностью переработанная коллекция классов (Collections Framework), включающая такие структуры данных, как `List`, `Set`, `Map`.
- Поддержка регулярных выражений.
- Появление Swing – библиотеки графического пользовательского интерфейса (GUI).
- Механизм прав доступа (Access Control).

## J2SE 1.3 (2000)
Этот релиз добавил ряд улучшений и новых возможностей:
- Улучшение производительности за счет оптимизации работы JIT-компилятора.
- Подключение звука через API Java Sound.
- Улучшения в поддержке XML.
- Новая система логирования (`java.util.logging`).

## J2SE 1.4 (2002)
Значительные улучшения:
- Поддержка исключительных ситуаций (Exceptions Chaining).
- Обработка утверждений (`assert`).
- Новые утилиты ввода-вывода (NIO).
- Возможность работы с регулярными выражениями через пакет `java.util.regex`.
- Улучшенная поддержка SSL/TLS.

## J2SE 5.0 (2004)
Этот релиз также известен как **Java 5**. Он принес множество значительных изменений:
- Генерализованные типы (Generics) – возможность создания универсальных типов данных.
- Аннотации (Annotations) – метаданные, добавляемые к коду для различных целей.
- Автоупаковка/распаковка примитивных типов (Autoboxing/Unboxing).
- Перечисления (Enums) – новые возможности для создания перечисляемых типов.
- Методы с переменным числом аргументов (Varargs).
- Улучшенные циклы (`for-each`) для итерации коллекций.
- Автоматическое управление ресурсами с помощью конструкции `try-with-resources`.

## Java SE 6 (2006)
Основные нововведения:
- Обновленный механизм управления памятью.
- Интеграция скриптового движка JavaScript (Rhino).
- Усовершенствования в работе с сетевыми соединениями.
- Новый набор инструментов для разработки GUI-приложений (SwingWorker).

## Java SE 7 (2011)
Важный релиз, содержащий следующие изменения:
- Добавлены строки в `switch` операторах.
- Упрощенный синтаксис для инициализации массивов (диамантный оператор `<>`).
- Поддержка строковых литералов в многострочном формате.
- Работа с бинарным представлением чисел (новый литерал для двоичных чисел).
- Появился новый интерфейс `java.nio.file` для работы с файловой системой.
- Многопоточность улучшилась благодаря классу `ForkJoinPool`.


## Java SE 8 LTS (2014)
Один из наиболее значимых релизов, который принес важные новшества:
- Лямбда-выражения – новая форма анонимных функций.
- Потоковые операции (Streams API) – упрощение обработки коллекций данных.
- Интерфейсы с методами по умолчанию (default methods).
- Дата и время (Date/Time API) – новая библиотека для работы с датами и временем.
- Повторная компиляция JIT-компилятором для повышения производительности.

  
## Java SE 9 (2017)
Релиз содержал значительные архитектурные изменения:
- Модульная система (JPMS) – введение модулей для организации кода.
- REPL-интерпретатор командной строки JShell.
- Оптимизация сборщика мусора (G1 стал дефолтным сборщиком).
- Пакет `ProcessHandle` для управления процессами.
- Конкатенация строк через метод `String::concat`.

Версия Java SE 9, выпущенная в сентябре 2017 года, стала одной из важнейших версий в истории развития языка. Она принесла значительное количество нововведений, включая модульную систему, инструменты для разработчиков и различные улучшения производительности. Рассмотрим эти изменения подробнее.

### 1. **Модульная система (Project Jigsaw)**
Самым значимым изменением в Java 9 стало внедрение модульной системы, известной как Project Jigsaw. Эта система позволяет разбивать большие приложения на модули, каждый из которых может содержать свои зависимости, а также управлять доступом между модулями. Основные аспекты модульной системы включают:

- **Модули**: Каждый модуль представляет собой отдельный компонент программы, имеющий четко определенные границы видимости и зависимостей. Это улучшает структуру проекта и облегчает поддержку больших приложений.
- **Файл module-info.java**: Этот файл описывает модуль, его экспортируемые пакеты и зависимости от других модулей. Например:
  ```java
  module com.example.myapp {
      requires java.sql;
      exports com.example.myapp.model;
  }
  ```
- **Платформенные модули**: Java Runtime Environment (JRE) теперь состоит из набора модулей, что делает её легче и гибче. Вы можете создавать собственные образы JRE, содержащие только те модули, которые необходимы вашему приложению.

### 2. **JShell (Read-Eval-Print Loop)**
JShell — это интерактивная среда исполнения Java-кода, которая позволяет разработчикам быстро тестировать фрагменты кода без необходимости писать полноценные программы. Это аналогично REPL в Python или Ruby. С помощью JShell вы можете вводить выражения, операторы и даже целые методы, видеть результаты сразу же после их выполнения и экспериментировать с различными возможностями языка.

Пример использования:
```shell
jshell> int x = 42;
x ==> 42

jshell> System.out.println("Hello, World!");
Hello, World!
```

### 3. **Улучшения в API**
- **Реактивные потоки (Flow API)**: В Java 9 добавлен новый API для работы с реактивными потоками данных, основанный на спецификации Reactive Streams. Это позволяет разрабатывать асинхронные приложения, реагирующие на события в реальном времени.
- **HTTP/2 Client**: Новый клиентский API для работы с протоколом HTTP/2, который поддерживает сжатие заголовков, мультиплексирование запросов и серверные push-сообщения.
- **API для работы с Process API**: Улучшено управление процессами, запущенными из Java-программ. Теперь можно получать информацию о состоянии процесса, управлять его завершением и отслеживать дочерние процессы.

### 4. **Улучшения в сборке мусора**
- **Сборщик мусора G1 стал дефолтным**: В предыдущих версиях Java сборщик мусора Parallel GC был основным выбором по умолчанию. В Java 9 G1 (Garbage First) стал новым стандартным сборщиком мусора, что должно улучшить производительность и предсказуемость работы программ.
- **Улучшена работа с памятью**: Были внесены изменения в управление памятью, что позволило уменьшить фрагментацию и повысить эффективность использования ресурсов.

### 5. **Другие важные изменения**
- **Конкатенация строк через метод String::concat**: Ранее конкатенация строк осуществлялась через оператор `+`. В Java 9 добавлена возможность использовать метод `concat()` для тех же целей.
- **Упрощённое создание коллекций**: В Java 9 были введены фабричные методы для создания неизменяемых коллекций. Например:
  ```java
  List<String> list = List.of("apple", "banana", "cherry");
  Set<Integer> set = Set.of(1, 2, 3);
  Map<String, Integer> map = Map.of("one", 1, "two", 2, "three", 3);
  ```
- **Удалены некоторые устаревшие компоненты**: Из Java 9 были удалены некоторые устаревшие технологии, такие как Applet API, CORBA и Java EE modules.

### Заключение
Java 9 привнесла значительные изменения, направленные на улучшение масштабируемости, производительности и удобства разработки. Введение модульной системы и JShell особенно важно для сообщества разработчиков, так как они открывают новые горизонты в создании крупных и сложных приложений.


## Java SE 10 (2018)
Основные нововведения:
- Локальные переменные типа `var` – автоматический вывод типа переменной.
- Параллельная полная сборка мусора G1.
- Инструменты для запуска единичных исходников файлов без необходимости их компилирования.

Java SE 10, выпущенная в марте 2018 года, хотя и не является такой значительной, как Java 9, всё же содержит несколько ключевых нововведений и улучшений. Вот подробный обзор этих изменений:

### 1. **Тип локальных переменных `var`**
Одним из главных нововведений в Java 10 стало появление ключевого слова `var` для объявления локальных переменных. Оно позволяет компилятору автоматически определять тип переменной на основе значения, которое ей присваивается. Это делает код короче и удобнее для чтения. Однако `var` можно использовать только для локальных переменных внутри методов, циклов и конструкторов.

Пример:
```java
var message = "Hello, World!";
var count = 42;
```

### 2. **Параллельный полный сборщик мусора G1**
G1 (Garbage First) стал дефолтным сборщиком мусора ещё в Java 9, но в Java 10 он был дополнительно оптимизирован для параллельной полной сборки мусора. Это означает, что сборка мусора происходит быстрее и эффективнее, что положительно сказывается на производительности приложений.

### 3. **Инструмент для запуска одиночных исходников файлов**
В Java 10 появился инструмент `jdk.internal.misc.FileUtils`, позволяющий запускать одиночный исходник Java-файла без предварительной компиляции. Это удобно при быстром прототипировании и тестировании небольших фрагментов кода.

Пример использования:
```bash
$ java MyClass.java
```

### 4. **Улучшения в API**
- **Улучшен класс `Optional`**: В Java 10 добавлено новое ограничение на использование метода `orElseThrow()`, чтобы предотвратить возможные ошибки.
- **Новый метод `copyOf` в классе `Collections`**: Метод `copyOf` создаёт неизменяемую копию списка, массива или другого коллекционного объекта.

### 5. **Оптимизация компиляции**
- **JIT-компиляция**: В Java 10 появилась возможность выполнять JIT-компиляцию на уровне отдельных методов, что ускоряет выполнение часто используемых частей кода.
- **Микрооптимизации**: Внесено множество мелких улучшений в работу JVM, направленных на повышение общей производительности.

### 6. **Поддержка Unicode 10**
Java 10 обновила поддержку стандарта Unicode до версии 10, что добавляет новые символы и эмодзи, а также улучшает совместимость с другими системами.

### 7. **Новые флаги JVM**
Добавлены новые флаги для настройки поведения JVM, такие как:
- `--class-path` (прежний путь к классам `-cp` устарел),
- `-XX:+UseContainerSupport` (для поддержки контейнеров Docker),
- `-Xlog` (улучшает ведение логов).

### Заключение
Хотя Java 10 не принесла столь радикальных изменений, как предыдущие версии, она всё равно сделала шаг вперёд в плане удобства разработки и производительности. Введение `var` значительно упростило написание кода, а улучшения в работе сборщика мусора и JIT-компиляции повысили общую скорость и стабильность приложений.


## Java SE 11 LTS (2018)
Этот релиз включил несколько важных изменений:
- HTTP-клиент с поддержкой HTTP/2.
- Запуск одиночного файла Java без предварительного компилирования.
- Улучшенный сборщик мусора ZGC.
- Набор стандартных классов для работы с файлами CSV.

Java SE 11, выпущенная в сентябре 2018 года, получила статус долгосрочной поддержки (LTS). Этот релиз включает в себя несколько важных изменений и улучшений, которые делают разработку на Java проще и удобнее. Давайте рассмотрим их подробнее.

### 1. **Удаление устаревшего кода**
В Java 11 было удалено несколько компонентов, которые считались устаревшими или больше не поддерживались:
- **Java EE и CORBA**: Эти модули были перемещены в сторонние проекты, поскольку они больше не являются частью основной платформы Java.
- **Applets**: Технология апплетов была окончательно удалена из Java, так как она потеряла актуальность с развитием веб-технологий.
- **Java Web Start**: Эта технология также была удалена, так как современные браузеры перестали поддерживать её.

### 2. **Новая версия лицензии**
Начиная с Java 11, Oracle изменила лицензию на свою версию JDK. Теперь доступна бесплатная версия OpenJDK, которую можно использовать в коммерческих проектах без ограничений. Это сделало Java более доступной для широкого круга пользователей.

### 3. **HTTP-клиент с поддержкой HTTP/2**
В Java 11 был включен новый HTTP-клиент, который поддерживает протокол HTTP/2. Это позволяет разработчикам создавать высокопроизводительные и эффективные сетевые приложения. Клиент также поддерживает WebSocket и HTTPS.

Пример использования нового клиента:
```java
HttpClient client = HttpClient.newBuilder().build();
HttpRequest request = HttpRequest.newBuilder()
        .uri(URI.create("https://example.com"))
        .GET()
        .build();
client.sendAsync(request, BodyHandlers.ofString())
        .thenApply(HttpResponse::body)
        .thenAccept(System.out::println)
        .join();
```

### 4. **Запуск одиночного файла Java без компиляции**
Эта функция была введена в Java 10, но в Java 11 она была доработана и стабилизирована. Теперь можно запустить одиночный Java-файл непосредственно из командной строки без необходимости предварительно компилировать его.

Пример использования:
```bash
$ java HelloWorld.java
```

### 5. **Набор стандартных классов для работы с файлами CSV**
В Java 11 был добавлен новый класс `CsvHelper`, который упрощает чтение и запись данных в формат CSV. Это полезно для работы с табличными данными и обмена информацией между разными системами.

Пример использования:
```java
import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Paths;
import java.util.List;

public class CsvExample {
    public static void main(String[] args) throws IOException {
        // Чтение CSV-файла
        List<String[]> rows = Files.readAllLines(Paths.get("data.csv"))
                .stream()
                .map(line -> line.split(","))
                .toList();
        
        // Вывод содержимого CSV-файла
        for (String[] row : rows) {
            for (String cell : row) {
                System.out.print(cell + "\t");
            }
            System.out.println();
        }
    }
}
```

### 6. **Улучшения в работе с памятью**
В Java 11 были внесены улучшения в управление памятью, что позволило сократить потребление оперативной памяти и увеличить производительность приложений. Например, был добавлен новый алгоритм выделения памяти Epsilon, который практически не потребляет ресурсы на управление памятью.

### 7. **Поддержка Unicode 11**
Java 11 обновила поддержку стандарта Unicode до версии 11, что добавляет новые символы и эмодзи, а также улучшает совместимость с другими системами.

### 8. **Улучшения в JVM**
- **З-GC (Z-Garbage Collector)**: В Java 11 был представлен новый сборщик мусора Z-GC, который предназначен для работы с большими объемами памяти (до нескольких терабайт) и минимизирует паузы при сборе мусора.
- **Flight Recorder**: Это инструмент для мониторинга и анализа производительности Java-приложений. Он позволяет собирать данные о работе JVM в реальном времени и анализировать их для выявления узких мест и проблем с производительностью.

### Заключение
Java 11 является важным релизом, который предлагает множество полезных улучшений и нововведений. Удаление устаревшего кода и переход на новую лицензию сделали платформу более современной и удобной для использования. Новые функциональные возможности, такие как HTTP-клиент с поддержкой HTTP/2 и набор классов для работы с CSV, расширяют возможности разработчиков. А улучшения в управлении памятью и JVM повышают производительность и надежность приложений.

## Java SE 12 (2019)
Основные нововведения:
- Переключатель для шаблона текста (Switch Expressions).
- Коллекции неизменяемых копий.
- Микрооптимизации для повышения производительности.

Java SE 12, выпущенная в марте 2019 года, добавила несколько интересных нововведений и улучшений, несмотря на то, что этот релиз не являлся долгосрочным (LTS). Давайте рассмотрим основные изменения более детально.

### 1. **Переключатели для шаблонов (Switch Expressions)**
Одной из самых заметных особенностей Java 12 стало введение переключателей для шаблонов (switch expressions). Это изменение позволяет использовать оператор `switch` не только для ветвлений, но и для возврата значений. Кроме того, в этом релизе появились стрелочные метки (`->`), которые заменяют блоки кода в случае простых операций.

Пример использования:
```java
int result = switch (day) {
    case MONDAY, FRIDAY, SUNDAY -> 6;
    case TUESDAY                -> 7;
```

Java SE 15, выпущенная в сентябре 2020 года, продолжила тенденцию внесения небольших, но значимых улучшений в язык и платформу Java. Этот релиз фокусировался на повышении производительности, удобства разработки и устранения устаревших компонентов. Рассмотрим основные изменения более подробно.

### 1. **Hidden Classes**
В Java 15 были введены скрытые классы (hidden classes), которые предназначены для использования в динамически загружаемых библиотеках и фреймворках. Скрытые классы могут быть созданы и использованы без регистрации в системе классов JVM, что делает их идеальными для временных или внутренних нужд.

Пример использования:
```java
var lookup = MethodHandles.lookup();

                       .getDeclaredConstructor().newInstance();
MethodType mt = MethodType.methodType(void.class);

Class<?> hc = (Class<?>) constructor.invokeExact();
```

### 2. **Sealed Classes**
В Java 15 продолжилось развитие концепции запечатанных классов (sealed classes), начатой в Java 14. Sealed-классы позволяют ограничить список подклассов, которые могут наследоваться от базового класса. Это помогает обеспечить контроль над иерархией классов и предотвращает нежелательное наследование.

Пример использования:
```java
public sealed class Animal permits Cat, Dog {
    ...
}

public final class Cat extends Animal {
    ...
}

public non-sealed class Dog extends Animal {
    ...
}
```

### 3. **Eden/Gen0 GC**
В Java 15 был введен новый алгоритм сбора мусора Eden/Gen0, который направлен на увеличение производительности путем оптимизации работы с молодыми объектами. Этот алгоритм уменьшает количество времени, затрачиваемого на сборку мусора, за счет более быстрого освобождения памяти для краткосрочных объектов.

### 4. **З-GC (Z-Garbage Collector)**
Продолжается разработка и улучшение сборщика мусора Z-GC, который предназначен для работы с большими объемами памяти и минимизацией пауз при сборе мусора. В Java 15 были внесены дополнительные оптимизации, повышающие его производительность и стабильность.

### 7. **Поддержка Unicode 15**
Java 15 обновила поддержку стандарта Unicode до версии 15, что добавляет новые символы и эмодзи, а также улучшает совместимость с другими системами.

### Заключение
Java 15 продолжает тренд на постепенное улучшение языка и платформы Java, делая акцент на производительности и удобстве разработки. Как и в предыдущих версиях, в Java 17 было удалено несколько устаревших компонентов и технологий:
- **Deprecated APIs**: Некоторые устаревшие API были помечены как deprecated, что указывает на необходимость их замены в будущем.
- **Finalization**: Механизм финализации был удалён, так как он считался неэффективным и потенциально опасным.

### 5. **Улучшения в JVM**
- **Сборщик мусора ZGC**: Продолжалось совершенствование сборщика мусора ZGC, предназначенного для работы с большими объёмами памяти и минимизацией пауз при сборе мусора.
- **Микрооптимизации**: Как и в предыдущих версиях, в Java 17 было внесено множество мелких улучшений в работу JVM, направленных на повышение общей производительности.

### 6. **Поддержка Unicode 17**
Java 17 обновила поддержку стандарта Unicode до версии 17, что добавляет новые символы и эмодзи, а также улучшает совместимость с другими системами.

Введение запечатанных классов, шаблонов для записей и новых предпросмотровых функций делает этот релиз привлекательным для разработчиков, стремящихся к созданию эффективных и безопасных приложений.

Конечно! Давайте подробно рассмотрим изменения, которые были внесены в Java SE 18, выпущенную в марте 2022 года. Эта версия включает несколько интересных улучшений и экспериментальных функций, хотя она и не является такой значительной, как предыдущие LTS-релизы.

### Основные изменения в Java SE 18

#### 1. **Simple Web Server**
Одной из наиболее заметных особенностей Java SE 18 стало добавление простого встроенного веб-сервера. Это средство предназначено для быстрого развертывания простых статических сайтов или тестового окружения без необходимости настройки сложных серверных решений. Веб-сервер позволяет разработчикам быстро разворачивать веб-приложения для тестирования и демонстрации.

Пример использования:
```java
import com.sun.net.httpserver.HttpServer;

public class SimpleWebServer {
    public static void main(String[] args) throws Exception {
        HttpServer server = HttpServer.create(new InetSocketAddress(8080), 0);
        server.createContext("/", new MyHttpHandler());
        server.start();
    }
}
```

#### 2. **UTF-8 by Default**
Начиная с Java SE 18, кодировка UTF-8 становится стандартной для файлов исходного кода. Это изменение направлено на унификацию и устранение проблем, связанных с использованием разных кодировок в проектах. Теперь разработчики могут быть уверены, что файлы будут корректно интерпретироваться независимо от платформы.

#### 3. **Internet-Address Resolution SPI**
Добавлен новый Service Provider Interface (SPI) для разрешения интернет-адресов. Это дает возможность заменять стандартные механизмы DNS-разрешения на альтернативные реализации, что полезно для специализированных приложений или сред, где требуется нестандартное поведение.

#### 4. **Запрет на использование анонимных классов в switch-выражении**
Теперь анонимные классы больше нельзя использовать в `switch`-выражениях. Это связано с тем, что в предыдущих версиях было возможно создать ситуации, когда анонимные классы конфликтовали с новыми функциями, такими как паттерн-матчинг. Это изменение улучшает читаемость и предсказуемость кода.

#### 5. **Обновленная спецификация языка (JEP 409)**
Включен ряд мелких исправлений и уточнений в спецификации языка Java. Эти изменения касаются уточнения поведения некоторых конструкций и устранения неоднозначностей.

#### 6. **Удаление устаревших функций**
Как обычно, в новой версии были удалены некоторые устаревшие функции и API, которые больше не поддерживаются. Например, была удалена поддержка SSLv3, так как этот протокол считается уязвимым.

#### 7. **Другие незначительные изменения**
Кроме вышеперечисленных основных изменений, в Java SE 18 были внесены различные мелкие улучшения и оптимизации, направленные на повышение производительности и удобства использования языка.

### Экспериментальные функции
В Java SE 18 также включены некоторые экспериментальные функции, которые могут стать частью будущих версий языка:

- **Паттерн-матчинг для switch** (второй превью). Продолжается работа над функцией паттерн-матчинга для операторов `switch`. Эта функция позволяет писать более компактный и выразительный код, особенно при работе с различными типами данных.

- **Шаблонные записи** (второй превью). Продолжаются эксперименты с шаблонными записями, которые позволяют создавать более гибкие структуры данных.

### Заключение
Хотя Java SE 18 не содержит радикальных изменений, она представляет собой важный шаг в эволюции языка. Введение встроенного веб-сервера, стандартизация UTF-8 и улучшение механизмов адресации делают разработку на Java еще удобнее и эффективнее.
